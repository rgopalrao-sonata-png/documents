# Manual Testing Guide: `nudge_dormant_enrolled_enterprise_learners`

## What This Command Does

Identifies enterprise learners who enrolled **7–13 days ago** but have **not started** their course (0% progress), then fires a Segment event per learner that triggers a Braze nudge email.

```
Snowflake query → filter dormant learners → emit Segment event → Braze sends email
```

---

| Requirement | How to Verify |

| Snowflake credentials configured | Check `SNOWFLAKE_*` vars in `.env` or Django settings |
| Segment write key configured | Check `SEGMENT_KEY` in Django settings |

## Step 3 — Dry Run (No Emails Sent)

This is the **safest starting point**. The `--no-commit` flag runs the full Snowflake query and logs all matched learners, but **does not** fire any Segment events.

```bash
docker compose exec app python manage.py \
  nudge_dormant_enrolled_enterprise_learners --no-commit
```

**What to look for in the output:**

```
# Good — found dormant learners
[Dormant Nudge]  Process started.
[Dormant Nudge] Segment event fired for nudge email ... LMS User Id: 12345, ...
[Dormant Nudge] Execution completed.

# Also good — query worked, no dormant learners in the 7–13 day window
[Dormant Nudge]  Process started.
[Dormant Nudge] Execution completed.

# Bad — something broke
Traceback ...
```

> **Note**: Even in `--no-commit` mode the log message says "Segment event fired" — this is just the logger printing what *would* be sent. No actual event is emitted.

## Step 4 — Inspect Raw Query Results

To see exactly what data the Snowflake query returns (before any event logic), run this in the Django shell:

```bash

```

Then paste:

```python
from enterprise_access.apps.core.management.commands.\
    nudge_dormant_enrolled_enterprise_learners import QUERY
from enterprise_access.apps.core.snowflake import fetch_all_query_results

results = list(fetch_all_query_results(QUERY))
print(f"Total dormant learner rows: {len(results)}")

# Print first 3 rows in a readable format
COLUMNS = [
    'external_id', 'org_name', 'course_title', 'enrollment_count',
    'min_effort', 'max_effort', 'weeks_to_complete', 'pacing_type',
    'pacing_subtitle', 'display_name', 'course_image',
    'lms_web_url', 'discussion_link', 'home_link'
]
for i, row in enumerate(results[:3]):
    print(f"\n--- Row {i + 1} ---")
    for col, val in zip(COLUMNS, row):
        print(f"  {col}: {val}")
```

**Interpreting results:**

| Outcome | Meaning |
|---------|---------|
| `Total dormant learner rows: 0` | No enrollments match the 7–13 day / 0% progress criteria — normal with sparse test data |
| Rows printed with populated fields | Query works end-to-end |
| Exception raised | Snowflake query error — check the SQL or credentials |

## Step 5 — Validate the Segment Event Payload

Before sending real events, verify the payload structure is correct by mocking `track_event`:

```bash
docker compose exec app python manage.py shell
```

```python
from unittest import mock
from django.core.management import call_command

with mock.patch('enterprise_access.apps.track.segment.track_event') as mock_track:
    call_command('nudge_dormant_enrolled_enterprise_learners')

print(f"track_event called {mock_track.call_count} time(s)")
for call in mock_track.call_args_list:
    user_id, event_name, props = call.args
    print(f"\nUser ID : {user_id}")
    print(f"Event   : {event_name}")
    print(f"Payload keys: {list(props.keys())}")
```

Expected payload keys:
```
EXTERNAL_ID, ORG_NAME, COURSE_TITLE, ENROLLMENT_COUNT,
MIN_EFFORT, MAX_EFFORT, WEEKS_TO_COMPLETE, PACING_TYPE,
PACING_SUBTITLE, DISPLAY_NAME, COURSE_IMAGE,
LMS_WEB_URL, DISCUSSION_LINK, HOME_LINK
```

## Troubleshooting

### No rows returned from Snowflake

The query filters enrollments from exactly 7–13 days ago. If your test database has no data in that window, the result is intentionally empty. To confirm data exists for a wider range:

```sql
-- Run in Snowflake Web UI or SnowSQL
SELECT
    DATE(lms_enrollment_created) as enroll_date,
    COUNT(*) as cnt
FROM enterprise.ent_base_enterprise_enrollment
WHERE COALESCE(course_progress, 0) = 0
AND consent_granted = True
GROUP BY 1
ORDER BY 1 DESC
LIMIT 20;
```

If rows exist outside the 7–13 day window, your test data is valid — it just may not fall in today's target range.

```bash

print('SEGMENT_KEY set:', bool(key))
"
```
