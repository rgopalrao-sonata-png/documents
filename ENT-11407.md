In a devstack LMS shell:

```bash

make lms-shell

# Inside the container:
from django.test.utils import override_settings
# or use the token endpoint directly:
```

Or via the LMS token endpoint:

```bash
curl -s -X POST http://localhost:18000/oauth2/access_token \
  -d "client_id=<client_id>&client_secret=<secret>&grant_type=client_credentials" \
  | python -m json.tool
```

Set it in your shell for all commands below:

```bash
TOKEN="<paste-jwt-here>"
```

## Endpoints

### 1. List Highlight Sets (GET)

```bash
curl -s "$BASE/?enterprise_uuid=$ENTERPRISE_UUID" \
```

### 2. Edit Highlight Title (POST)

**POST** `/{uuid}/edit-highlight-title/`

#### ✅ Success — rename the set

```bash
curl -s -X POST "$BASE/$SET_UUID/edit-highlight-title/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "Renamed Highlight Set"}' | python -m json.tool
```

**Expected:** 200, full HighlightSet object with `"title": "Renamed Highlight Set"`.

#### ❌ Validation — title too long (>60 chars)

```bash
curl -s -X POST "$BASE/$SET_UUID/edit-highlight-title/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "This title is way too long and should definitely exceed sixty characters easily"}' \
  | python -m json.tool
```

**Expected:** 400, `{"Error": "title cannot exceed 60 characters"}`.

#### ❌ Validation — missing title

```bash
curl -s -X POST "$BASE/$SET_UUID/edit-highlight-title/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}' | python -m json.tool
```

**Expected:** 400, `{"Error": "Missing title parameter"}`.

#### ❌ Not found — random UUID

```bash
curl -s -X POST "$BASE/00000000-0000-0000-0000-000000000099/edit-highlight-title/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "Ghost"}' | python -m json.tool
```

**Expected:** 403 or 404.

### 3. Edit Highlight — Combined PATCH (NEW)

**PATCH** `/{uuid}/edit-highlight/`

Atomically updates title + adds/removes content in one request.

#### ✅ Title only

```bash
curl -s -X PATCH "$BASE/$SET_UUID/edit-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "Title Only Update"}' | python -m json.tool
```

**Expected:** 200, `added_content_keys: []`, `removed_content_keys: []`.

#### ✅ Add a new course

```bash
curl -s -X PATCH "$BASE/$SET_UUID/edit-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"add_content_keys": ["course-v1:edX+NEW101+2023"]}' | python -m json.tool
```

**Expected:** 200. If the content key exists in the catalog → in `added_content_keys`; if not → in `ignored_content_keys`.

#### ✅ Remove a course

```bash
curl -s -X PATCH "$BASE/$SET_UUID/edit-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"remove_content_keys": ["course-v1:edX+DemoX+Demo_Course"]}' | python -m json.tool
```

**Expected:** 200, `removed_content_keys: ["course-v1:edX+DemoX+Demo_Course"]`, course gone from `highlight_set.highlighted_content`.

#### ✅ Combined — title + add + remove in one shot

```bash
curl -s -X PATCH "$BASE/$SET_UUID/edit-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Combined Update",
    "add_content_keys": ["course-v1:edX+NEW101+2023"],
    "remove_content_keys": ["course-v1:edX+CS101+2023"]
  }' | python -m json.tool
```

**Expected:** 200, title changed, CS101 removed, NEW101 added (or ignored if not in catalog).

#### ❌ Validation — no fields provided

```bash
curl -s -X PATCH "$BASE/$SET_UUID/edit-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}' | python -m json.tool
```

**Expected:** 400, `{"Error": "At least one of title, add_content_keys, or remove_content_keys must be provided."}`.

#### ❌ Validation — title too long

```bash
curl -s -X PATCH "$BASE/$SET_UUID/edit-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "This title is way too long and should definitely exceed sixty characters easily"}' \
  | python -m json.tool
```

**Expected:** 400, `{"Error": "title cannot exceed 60 characters"}`.

### 4. Add Content (POST)

**POST** `/{uuid}/add-content/`

```bash
curl -s -X POST "$BASE/$SET_UUID/add-content/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content_keys": ["course-v1:edX+DemoX+Demo_Course"]}' | python -m json.tool
```

**Expected:** 201, keys split across `added_content_keys` / `existing_content_keys` / `ignored_content_keys`.

---

### 5. Remove Content (POST)

**POST** `/{uuid}/remove-content/`

```bash
curl -s -X POST "$BASE/$SET_UUID/remove-content/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content_keys": ["course-v1:edX+DemoX+Demo_Course"]}' | python -m json.tool
```

**Expected:** 201, removed key in `removed_content_keys`, not in `highlight_set.highlighted_content` anymore.

### 6. Toggle Favorite (POST)

**POST** `/{uuid}/toggle-favorite-highlight/`

```bash
# Favorite ON
curl -s -X POST "$BASE/$SET_UUID/toggle-favorite-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content_uuid": "0bb942ea-73a8-444d-a595-77870cbf97d3", "favorite": "true"}' | python -m json.tool

# Favorite OFF
curl -s -X POST "$BASE/$SET_UUID/toggle-favorite-highlight/" \
  -H "Authorization: JWT $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content_uuid": "0bb942ea-73a8-444d-a595-77870cbf97d3", "favorite": "false"}' | python -m json.tool
```

**Expected:** 201 `{}` both times. Verify in DB that `is_favorite` flipped.
